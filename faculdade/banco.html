Anotações

O banco de dados é um conjunto organizado de informações/dados, geralmente armazenadas em um sistema de computador e controladas por um sistema de gerenciamento de banco de dados. Tive dificuldades durante esta matéria principalmente ao realizar diagrama entidade-relacionamento, porém gostei da parte de programação, apesar de ter muitos comandos para lembrar.

Programação:

Toda a programação foi feita utilizando a linguagem SQL (Structured Query Language) e usando o software PostgreSQL.

Modelagem de Dados

Modelo Entidade-Relacionamento

Representação visual de um banco de dados utilizando diagramas.

Componentes
  • Entidades;
    • Representam objetos ou conceitos do mundo real com existência independente;
  • Atributos;
    • Características das entidades;
  • Relacionamentos;
    • Definem como as entidades estão associadas entre si.

Exemplo de diagrama
  • Entidades:
    • Cursos;
    • Alunos.
  • Atributos:
    • Cursos:
      • ID_Curso;
      • Nome_Curso.
    • Alunos:
      • Id_Aluno;
      • Dt. nasc;
      • Nome.
  • Relacionamentos:
    • Turmas.

Dados

Dados estruturados

  • Dados organizados em um formato fixo, facilita pesquisa, fácil de extrair informação.


    • Dados não estruturados

  • Dados não organizados ou não têm um formato predefinido, difícil extração.


    • Dados semi estruturados

  • Dados que não são organizados em tabelas, mas possuem tags ou marcadores para identificar os elementos.


    • Modelo de negócios
    • Empresas - Cliente;
    • Dados necessários;
      • Funcionário: nome, departamento, cpf;
      • Departamento: nome, código do departamento.
    • Regras de negócio;
      • Um funcionário só pode pertencer a um departamento;
      • Um departamento pode ter vários funcionários.

    Modelo conceitual

    • Regras
      • Em um relacionamento com cardinalidade 1:1 - 1:N, a chave estrangeira deve ficar sempre no lado “N”;
      • Em um relacionamento com cardinalidade 1:N-1:N, o relacionamento se torna uma entidade que irá conter ambas FK.
    • Diagrama Entidade Relacionamento (DER);
    • Todo conjunto de dados é uma entidade;
    • Entidade é representada por um retângulo;
    • Código dentro da entidade é chamada de *Primary Key (PK)*, ele precisa ser unique (único) e not null (não estar vazio);
    • Chave candidata: segunda opção do código,

    Comandos DQL 
    
    DROP TABLE product; 
    DROP TABLE client CASCADE;

    CREATE TABLE clients (
        idclient SERIAL PRIMARY KEY,
        cpf varchar(14),
        firstname text,
        lastname text,
        address text
    );

    CREATE TABLE products (
        idproduct SERIAL PRIMARY KEY,
        idclient int,
        CONSTRAINT fk_client FOREIGN KEY (idclient) REFERENCES clients (idclient),
        productname varchar(50),
        brand varchar(20),
        category varchar(20),
        discount numeric,
        price numeric
    );

    Comandos Cross Join 
    
SELECT * FROM clients,products --Cliente vai aparecer primeiro
SELECT * FROM products,clients --Produto vai aparecer primeiro
SELECT * FROM clients CROSS JOIN products --Forma alternativa

-- Inner Join (relaciona por PK/FK)
-- Mostra todas as colunas
SELECT * FROM clients INNER JOIN products
ON clients.idclient = products.idclient --Nome da tabela.nome da coluna

-- Mostra as colunas específicas (firstname e productname)
SELECT firstname,productname FROM clients INNER JOIN products
ON clients.idclient = products.idclient

-- Buscar só de um cliente
SELECT firstname,productname FROM clients INNER JOIN products
ON clients.idclient = products.idclient
WHERE firstname = 'Rodrigo' OR firstname ='rodrigo'

--Comandos alternativos
SELECT * FROM clients INNER JOIN products USING (idclient)

SELECT * FROM clients NATURAL INNER JOIN products--Entende que vai usar PK/FK
